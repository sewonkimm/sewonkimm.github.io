## 왜 사용하나요?

- 리액트 프레임워크이기에 리액트 개발자의 러닝커브가 낮다.
  - 문서와 예제코드도 잘 제공된다.
  - 커뮤니티가 많다.
- 간단한 기능은 매우 빠르게 개발이 가능하다.
  - 초반 세팅에 들어가는 시간이 줄어들고, 확장성도 높다.
  - 불필요한 세팅을 최소한으로 줄여주고 코드 자체에 집중할 수 있게 해주는 환경을 제공한다.
- SSR을 지원한다. 
  - 화면 깜빡임이 없다.
  - SEO 적용에 유리하다.
- 비슷한 프레임워크로는 Remix, Gatsby, Svelte 등이 있다.
